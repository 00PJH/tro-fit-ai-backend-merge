## 1. 핵심 결론: 처음 자세(중립)와 마지막 자세(최대) 둘 다 필요한가?

### ✅ 결론: **두 자세 모두 촬영이 필요합니다**

---

### 이유: ROM의 정의와 계산 방식

**ROM (Range of Motion, 관절 가동범위)** 의 정의는:

```
ROM = 최대 동작각 − 중립(해부학적 기준) 각도
```

임상 참고 자료(국민연금 기준, 관절가동범위_1.png)에서 어깨 외전(Shoulder Abduction):
- **정상 운동범위: 0° ~ 180°**
- 측정 기준: 팔을 옆구리에 붙인 자세(0°)에서 최대로 들어올린 자세(최대°)

즉, ROM은 항상 **시작점(0° 기준 자세)** 이 존재합니다.

---

### 왜 중립 자세를 "생략할 수 없는가"

MediaPipe가 계산하는 각도는 **절대 임상 각도가 아닙니다**. 벡터 내적 기반의 상대 각도입니다.

```python
# 어깨 각도: Elbow → Shoulder → Hip 사이각
shoulder_angle = calculate_angle_3d(elbow, shoulder, hip)
```

| 자세 | 계산되는 shoulder_angle | 임상 의미 |
|------|------------------------|-----------|
| 팔 옆구리 붙임 (중립) | ≈ **175° ~ 178°** | 임상 0° (해부학적 기준) |
| 팔 90° 외전 | ≈ **90° ~ 95°** | 임상 90° |
| 팔 최대 외전 | ≈ **40° ~ 70°** | 임상 약 120°~170° |

**핵심 문제**: 중립 자세에서의 각도가 사람마다 다릅니다.

- 키가 큰 사람: 팔과 고관절의 상대 위치가 달라 중립각이 177°
- 키가 작은 사람: 중립각이 173°
- 카메라 각도가 틀어지면: 중립각이 170°
- 팔을 약간 앞으로 기울인 채로 서면: 중립각이 165°

이 개인별 편차가 최대 15° 이상 발생할 수 있으며, 이를 무시하면:

```
잘못된 계산 (중립 생략, 180° 가정):
  ROM ≈ 180° − final_angle
  예: 180° − 65° = 115°  ← 실제 ROM과 최대 15° 오차 가능

정확한 계산 (중립 포함):
  ROM = neutral_angle − final_angle
  예: 175° − 65° = 110°  ← 개인 맞춤 기준점 적용
```

> **임상 등급 정확도 요구 시**: 반드시 중립 자세 캡처 필요  
> **Tro-Fit의 선택**: 재활/임상 보조 앱 → **중립 자세 캡처 포함**

## 2. 측정 흐름 설계

### 전체 타임라인

```
[Phase 1: 포지셔닝]     [Phase 2: 중립 캡처]    [Phase 3: 최대 자세 유지 + 캡처]
0s ─────────── ?s  │  ?s ────── 1s  │  1s ─────────────────────────── 6s
 실시간 자세 가이드  │  중립 이미지    │  동작 → 3초 유지 → 마지막 2초에서 이미지 2장 캡처
                   │  1장 캡처       │
```

---

### Phase 1: 포지셔닝 (실시간 가이드)

> **UX 목표**: 전신이 카메라에 들어오고, 측정할 관절의 visibility가 확보될 때까지 실시간 피드백 제공.

**화면 안내 문구 + TTS**:

```
"정면을 바라보고 서주세요. 전신이 화면에 나오도록 해주세요."
```

**실시간 피드백 (MediaPipe visibility 기반)**:

| 조건 | 피드백 문구 |
|------|-----------|
| shoulder.visibility < 0.5 | "어깨가 보이지 않습니다. 뒤로 조금 물러나 주세요." |
| hip.visibility < 0.5 | "전신이 나오지 않습니다. 카메라를 더 멀리 놓아주세요." |
| 왼쪽 어깨가 화면 끝에 걸림 | "오른쪽으로 이동해 주세요." |
| 오른쪽 어깨가 화면 끝에 걸림 | "왼쪽으로 이동해 주세요." |
| 모든 관절 visibility >= 0.7 | "✅ 준비됐습니다. 측정을 시작합니다." |

---

### Phase 2: 중립 자세 캡처 (Static Image, 1장)

> **UX 목표**: 해부학적 중립 자세(팔을 몸에 붙인 자세)를 1장 캡처하여 개인 기준 각도를 설정.

**화면 안내 문구 + TTS**:

```
"양팔을 옆구리에 자연스럽게 붙이세요."
[카운트다운 3... 2... 1...]
[📸 찰칵 — 중립 자세 캡처 완료]
```

**캡처 자동 트리거 조건**:
- 모든 관련 관절 `visibility >= 0.65`
- 어깨 각도가 안정화됨 (최근 10프레임 표준편차 < 2°)
- 카운트다운 후 자동 캡처 (사용자가 버튼 누를 필요 없음)

**저장 데이터**:
```json
{
  "phase": "neutral",
  "joint": "shoulder_abduction",
  "left_shoulder_angle": 176.3,
  "right_shoulder_angle": 174.8,
  "visibility_left": 0.91,
  "visibility_right": 0.88
}
```

### Phase 3: 최대 자세 유지 + 캡처 (5초 비디오 → 마지막 2초 추출)

> **UX 목표**: 최대 동작 자세를 3초간 유지하게 하고, 가장 안정된 구간(마지막 2초)에서 최적 프레임 2장을 추출하여 더 나은 것을 사용.

**화면 안내 문구 + TTS**:

```
"양팔을 옆으로 최대한 들어올리세요."
[동작 시작 감지됨]
"자세를 3초간 유지해 주세요."
[3... 2... 1... ✅ 측정 완료]
```

**5초 비디오 타임라인 (30 FPS 기준)**:

```
0s ──── 1s ──── 2s ──── 3s ──── 4s ──── 5s
│              │         │       │       │
│ 동작 시작    │         │       │       │
│ (움직임 감지)│         │  ★캡처1│  ★캡처2│
│              │         │ (3~4s) │ (4~5s) │
│<─── 흐트러진 구간 ────>│<── 안정 구간 ──>│
```

**프레임 추출 전략**:

```
전체 5초 비디오 (예: 30 FPS = 150 프레임)

캡처 구간 A: 3~4초 구간 → 중간 프레임 1장 (약 frame #105)
캡처 구간 B: 4~5초 구간 → 중간 프레임 1장 (약 frame #135)

최종 선택 기준:
  1. 관련 관절 visibility 합산이 높은 것
  2. 동점이면 측정 각도가 더 큰 것 (더 많이 벌린 자세 = 더 정확한 최대값)
```

**선택 알고리즘 (Python 의사코드)**:

```python
def select_best_frame(frame_a: dict, frame_b: dict, joint: str) -> dict:
    """
    두 후보 프레임 중 ROM 측정에 더 적합한 프레임을 선택합니다.
    """
    vis_a = get_joint_visibility(frame_a, joint)
    vis_b = get_joint_visibility(frame_b, joint)
    
    # visibility 차이가 0.1 이상이면 visibility 우선
    if abs(vis_a - vis_b) > 0.1:
        return frame_a if vis_a > vis_b else frame_b
    
    # visibility가 비슷하면 각도가 더 큰 쪽 선택 (더 벌린 자세)
    angle_a = get_angle(frame_a, joint)
    angle_b = get_angle(frame_b, joint)
    return frame_a if angle_a > angle_b else frame_b
```

## 3. ROM 계산 공식

### 어깨 외전 예시

```
중립 자세 캡처 결과:
  neutral_angle = 175.3°   (팔 붙인 자세에서 Elbow→Shoulder→Hip 각도)

최대 자세 캡처 결과 (더 좋은 프레임 선택):
  max_angle = 62.7°        (팔 최대로 든 자세에서 동일 벡터 각도)

ROM = neutral_angle − max_angle
    = 175.3° − 62.7°
    = 112.6°

임상 정상 범위: 0° ~ 180°
결과: 112.6° → "정상 범위의 62.6%"
```

### 관절별 벡터 정의 (MediaPipe 기반)

| 관절 | 벡터 구성 | ROM 방향 |
|------|----------|---------|
| 어깨 외전 (Shoulder Abduction) | Elbow → **Shoulder** → Hip | neutral − max |
| 고관절 굴곡 (Hip Flexion) | Shoulder → **Hip** → Knee | max − neutral |
| 무릎 굴곡 (Knee Flexion) | Hip → **Knee** → Ankle | max − neutral |
| 팔꿈치 굴곡 (Elbow Flexion) | Shoulder → **Elbow** → Wrist | max − neutral |
| 발목 배측굴곡 (Ankle Dorsiflexion) | Knee → **Ankle** → Foot Index | max − neutral |

> **방향 주의**: 어깨 외전은 각도가 "줄어드는" 방향이 ROM. 나머지는 "늘어나는" 방향.

## 4. 왜 "마지막 2초"가 올바른 전략인가

### 동작 구간별 특성

```
[전형적인 어깨 외전 5초 동작 - 각도 변화]

175°│▓▓▓▓
    │    ▓▓
    │      ▓▓▓▓▓
 80°│          ▓▓▓▓▓▓▓▓▓▓▓▓ ← 안정 구간 (±2°)
    └─────────────────────────
    0s  1s  2s  3s  4s  5s

흐트러짐 구간 (0~3s):
  - 동작 시작 → 최고점 도달까지 변화 중
  - 근육 긴장/보상 운동 → 각도 불안정

안정 구간 (3~5s):
  - 최대 관절 가동범위에서 자세 유지
  - 각도 변화 ±2° 이내
  - 이 구간이 실제 ROM 측정 목표
```

### 기술적 장점

1. **노이즈 제거**: 동작 시작 시 관성/흔들림으로 인한 일시적 오버슈트 제거
2. **신뢰도 검증**: 두 프레임(3~4s, 4~5s)의 각도 차이가 5° 이내 → "안정된 자세" 확인
3. **연산 효율**: 전체 150프레임 처리 대신 12~15프레임만 처리 → 모바일 배터리 절약
4. **사용자 경험**: 5초 측정 = 짧고 명확, 사용자 피로 최소화

## 5. 안정성 검증 (신뢰도 게이트)

```python
angle_diff = abs(angle_frame_a - angle_frame_b)

if angle_diff <= 3.0:
    confidence = "HIGH"    # 안정된 자세 유지 확인
elif angle_diff <= 7.0:
    confidence = "MEDIUM"  # 약간 흔들림 있었음 → 더 좋은 프레임 선택
else:
    confidence = "LOW"     # 자세 유지 실패 → 재측정 권고
    # TTS: "자세가 흔들렸습니다. 다시 측정해 주세요."
```

## 7. 요약 결정 테이블

| 항목 | 결정 | 이유 |
|------|------|------|
| 중립 자세 캡처 여부 | **필요** | 개인 기준점 없으면 최대 15° 오차 가능 |
| 중립 자세 캡처 방식 | **정지 이미지 1장** | 고정 자세 → 비디오 불필요 |
| 최대 자세 캡처 방식 | **5초 비디오 → 마지막 2초 구간** | 안정 구간 확보, 흔들림 제거 |
| 후보 프레임 수 | **2장** (3~4s 중간, 4~5s 중간) | 하나 실패해도 다른 것으로 대체 |
| 최종 프레임 선택 기준 | **visibility 우선, 동점이면 각도 우선** | 신뢰도가 정확도보다 중요 |
| 재측정 트리거 | **두 프레임 각도 차 > 7°** | 자세 유지 실패 판정 |
| ROM 계산 방식 | **개인 중립각 − 최대각** | 절대각도 가정 대비 정확도 향상 |
| 좌표 기준 | **world_landmarks (미터 단위)** | 카메라 거리 영향 제거 |
